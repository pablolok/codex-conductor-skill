# .NET and Desktop App Styleguide

## Engineering Alignment

- Respect the repository's .NET, Avalonia, MVVM, and DI rules from `AGENTS.md`.
- Keep testing and verification aligned with the repository's mandatory .NET validation steps.
- Prefer changes that preserve cross-platform behavior on Windows, macOS, and Linux.

## Track Guidance

- Specs should identify the user-visible outcome or runtime behavior being changed.
- Plans should call out affected solution areas, release-mode verification, and test coverage expectations.
