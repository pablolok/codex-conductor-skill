# Brownfield Scan

When the repository is brownfield:

1. Ask permission for a read-only scan before analyzing the codebase.
2. Respect the repository's ignore files where possible.
3. Prioritize:
   - `README.md`
   - manifests and solution files
   - `src/` shape
   - repository guidance files such as `AGENTS.md`
4. Infer:
   - product purpose
   - stack
   - architecture shape
   - established style or workflow signals
5. Use questions only to fill genuine gaps or choose between meaningful preferences.
